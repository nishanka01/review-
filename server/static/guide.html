OpenEnv Simulation Guide

CodeReview Reinforcement Learning

01 The Objective

This environment acts as a "gym" for AI agents. The goal is to train an agent that can analyze source code, identify specific weaknesses (bugs, inefficiencies, or security flaws), and apply improvements to maximize its Reward Score.

02 Simulation Tasks

Easy: Naming Logic

Goal: Change cryptic variables (a, b) to descriptive ones (width, height).

Grader: Regex-based name validation.

Medium: Performance Optimization

Goal: Convert list-based lookups O(n) to set-based lookups O(1).

Grader: AST-based complexity analysis.

Hard: Security Patching

Goal: Fix SQL injection vulnerabilities using parameterized queries.

Grader: Static analysis for safe DB patterns.

03 Core Workflow

  • Reset: Clears the environment and loads the "broken" code for a task.
  • Step: The agent submits a code change (Action).
  • Observation: The agent receives the updated code.
  • Reward: A float (0.0 to 1.0) indicating how much the code improved.

04 How to Interact

You can use the built-in dashboard at http://localhost:8080 to manually test the tasks. To train an agent, connect to the following API endpoints:

POST /reset?task_id=task_1

POST /step (JSON: {action_type: "suggest_fix", content: "..."})

GET /state (Retrieves full trajectory and history)
